## 二、项目改进建议

### 2.1 紧急改进（优先级最高）

#### 1️⃣ 实现真实的 rosbridge WebSocket 连接

**现状**：`useRosbridge.ts` 中 WebSocket 连接被注释，使用 `setTimeout` 模拟连接成功。

**改进方案**：
```typescript
// useRosbridge.ts - 真实连接实现
const ws = new WebSocket('ws://localhost:9090');
ws.onopen = () => {
  setRosConnection('connected');
  // 订阅 ROS2 话题
  subscribeTopic(ws, '/vision/detection_text', 'std_msgs/String');
  subscribeTopic(ws, '/chat_output', 'std_msgs/String');
};
ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  handleRosMessage(msg);
};
```

#### 2️⃣ 实现真实的摄像头视频流 + 检测框叠加

**现状**：`useDetections.ts` 使用 `generateMockDetections()` 生成随机检测数据。

**改进方案**：
- 通过 rosbridge 订阅 `/camera/image_raw` 话题获取真实视频帧
- 通过 rosbridge 订阅 `/vision/detection` 获取真实检测结果
- 在 canvas 上绘制真实的检测边界框
- 实现检测框坐标与实际视频尺寸的适配

#### 3️⃣ 实现真实的 AI 对话

**现状**：`useChat.ts` 使用硬编码的规则匹配生成回复。

**改进方案**：
- 通过 rosbridge 向 `/chat_input` 发布用户消息
- 通过 rosbridge 订阅 `/chat_output` 获取 AI 回复
- 支持流式输出（SSE/WebSocket 流式响应）

### 2.2 中期改进

#### 4️⃣ 对话历史管理（后端）
- 实现多轮对话上下文管理，支持会话历史缓存
- 添加 `/chat/history` 话题发布对话历史
- 实现滑动窗口机制，控制上下文长度

#### 5️⃣ 错误处理增强
- 添加 Ollama 请求重试机制（指数退避）
- 实现节点健康检查心跳（`/health` 话题）
- 添加 Ollama 服务可用性检测

#### 6️⃣ 视觉识别优化
- 实现多模型级联策略（YOLO 快速检测 + Ollama 深度分析）
- 添加滑动窗口去重，避免连续帧重复检测
- 支持检测结果可视化（在图像上绘制边界框）

### 2.3 长期改进

#### 7️⃣ 语音交互
- 集成语音识别（STT）节点
- 集成语音合成（TTS）节点
- 实现完整的语音对话流程

#### 8️⃣ 性能监控
- 添加 Prometheus 指标暴露
- 实现结构化日志存储
- 可视化推理延迟和资源使用

## 三、下一阶段前端界面设计方案

### 3.1 设计目标

构建一个**功能完整、视觉精美、交互流畅**的 Web 前端界面，实现：

1. **实时摄像头画面显示** - 通过 rosbridge 获取真实视频流
2. **实时对象检测与统计** - 显示 YOLO 检测结果，实时统计画面中的对象
3. **智能问答** - 基于实时摄像头画面的 AI 问答交流
4. **摄像头控制** - 开启/关闭摄像头，支持无摄像头的纯文本聊天

### 3.3 页面详细设计

#### 3.3.1 摄像头画面面板（CameraPanel）

**功能需求**：
- 实时显示摄像头视频流（通过 rosbridge 订阅 `/camera/image_raw`）
- 在视频上叠加 YOLO 检测边界框（通过 rosbridge 订阅 `/vision/detection`）
- 显示视频信息：分辨率、帧率、检测延迟
- 支持全屏模式
- 摄像头开启/关闭控制

**交互流程**：
```
用户点击"开启摄像头"
  → 请求浏览器摄像头权限 (getUserMedia)
  → 显示本地视频预览
  → 通过 rosbridge 订阅 /camera/image_raw 和 /vision/detection
  → 在 canvas 上绘制检测框叠加层

用户点击"关闭摄像头"
  → 停止本地视频流
  → 取消 rosbridge 订阅
  → 清空检测框
```

**状态管理**：
- `isCameraOn: boolean` - 摄像头开关状态
- `videoStream: MediaStream | null` - 本地视频流
- `detections: Detection[]` - 当前检测结果
- `fps: number` - 实时帧率
- `latency: number` - 检测延迟

#### 3.3.2 对象实时统计面板（StatsPanel）

**功能需求**：
- 实时显示检测到的对象列表（名称、数量、置信度）
- 饼图展示对象分布比例
- 柱状图展示各对象数量对比
- 每个对象用不同颜色标识
- 摄像头关闭时显示"暂无数据"

**数据流**：
```
/vision/detection (Detection2DArray)
  → 解析检测结果
  → 按 class_name 聚合统计
  → 更新对象列表和图表
```

**统计指标**：
- 当前画面中各类对象数量
- 对象总数
- 各类对象占比

#### 3.3.3 AI 对话面板（ChatPanel）

**功能需求**：
- 聊天消息列表（用户消息、AI 回复、系统消息）
- 消息输入框（支持 Enter 发送、Shift+Enter 换行）
- Markdown 渲染（AI 回复支持富文本）
- 打字指示器（AI 正在回复时显示）
- 自动滚动到最新消息
- 视觉模式/文本模式切换指示
- 摄像头开启时，AI 回复可引用检测结果

**交互流程**：
```
用户输入消息 → 发送
  → 通过 rosbridge 发布到 /chat_input
  → 显示用户消息
  → 显示打字指示器
  → 通过 rosbridge 订阅 /chat_output
  → 收到 AI 回复
  → 隐藏打字指示器
  → 显示 AI 回复（Markdown 渲染）
```

**消息类型**：
| 类型 | 图标 | 样式 | 说明 |
|------|------|------|------|
| 用户消息 | 👤 | 蓝色气泡，右对齐 | 用户输入的内容 |
| AI 回复 | 🤖 | 深色气泡，左对齐 | AI 生成的回复，支持 Markdown |
| 系统消息 | ℹ️ | 灰色居中 | 系统提示、状态变更通知 |

**视觉模式交互**：
- 摄像头开启时，AI 自动获取检测结果作为上下文
- 用户可询问"我现在面前有什么？"、"有几个杯子？"等
- AI 回复中引用检测到的对象及其位置

#### 3.3.4 状态栏（StatusBar）

**功能需求**：
- 显示各节点运行状态（ROS Bridge、摄像头、YOLO、Ollama、LLM）
- 显示性能指标（帧率、检测延迟）
- 状态颜色指示（绿色=正常、黄色=连接中、红色=异常、灰色=停止）

**状态定义**：
| 节点 | 正常 | 异常 | 停止 |
|------|------|------|------|
| ROS Bridge | 🟢 connected | 🔴 disconnected/error | ⚫ - |
| 摄像头 | 🟢 connected | 🔴 error | ⚫ disconnected |
| YOLO | 🟢 running | 🔴 error | ⚫ stopped |
| Ollama | 🟢 connected | 🔴 error | ⚫ disconnected |
| LLM | 🟢 running | 🔴 error | ⚫ stopped |

#### 3.3.5 设置对话框（SettingsDialog）

**功能需求**：
- Ollama 模型选择（下拉列表）
- 检测置信度阈值调节（滑块）
- ROS Bridge WebSocket 地址配置
- 系统信息展示（版本、框架、检测引擎）

| `/vision/trigger` | `std_msgs/String` | 发布 | 手动触发检测 |
